# AWS::DataBrew::Job DatabaseOutput

Represents a JDBC database output object which defines the output destination for
a DataBrew recipe job to write into.

## Syntax

To declare this entity in your CloudFormation template, use the following syntax:

### JSON

```json

{
  "DatabaseOptions" : DatabaseTableOutputOptions,
  "DatabaseOutputMode" : String,
  "GlueConnectionName" : String
}

```

### YAML

```yaml

  DatabaseOptions:
    DatabaseTableOutputOptions
  DatabaseOutputMode: String
  GlueConnectionName: String

```

## Properties

`DatabaseOptions`

Represents options that specify how and where DataBrew writes the database output
generated by recipe jobs.

_Required_: Yes

_Type_: [DatabaseTableOutputOptions](https://docs.aws.amazon.com/AWSCloudFormation/latest/TemplateReference/aws-properties-databrew-job-databasetableoutputoptions.html)

_Update requires_: [No interruption](../userguide/using-cfn-updating-stacks-update-behaviors.md#update-no-interrupt)

`DatabaseOutputMode`

The output mode to write into the database. Currently supported option: NEW\_TABLE.

_Required_: No

_Type_: String

_Allowed values_: `NEW_TABLE`

_Update requires_: [No interruption](../userguide/using-cfn-updating-stacks-update-behaviors.md#update-no-interrupt)

`GlueConnectionName`

The AWS Glue connection that stores the connection information for the
target database.

_Required_: Yes

_Type_: String

_Minimum_: `1`

_Maximum_: `255`

_Update requires_: [No interruption](../userguide/using-cfn-updating-stacks-update-behaviors.md#update-no-interrupt)
