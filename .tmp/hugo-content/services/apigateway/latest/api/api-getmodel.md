# GetModel

Describes an existing model defined for a RestApi resource.

## Request Syntax

```nohighlight

GET /restapis/restapi_id/models/model_name?flatten=flatten HTTP/1.1

```

## URI Request Parameters

The request uses the following URI parameters.

**[flatten](#API_GetModel_RequestSyntax)**

A query parameter of a Boolean value to resolve ( `true`) all external model references and returns a flattened model schema or not ( `false`) The default is `false`.

**[model\_name](#API_GetModel_RequestSyntax)**

The name of the model as an identifier.

Required: Yes

**[restapi\_id](#API_GetModel_RequestSyntax)**

The RestApi identifier under which the Model exists.

Required: Yes

## Request Body

The request does not have a request body.

## Response Syntax

```nohighlight

HTTP/1.1 200
Content-type: application/json

{
   "contentType": "string",
   "description": "string",
   "id": "string",
   "name": "string",
   "schema": "string"
}
```

## Response Elements

If the action is successful, the service sends back an HTTP 200 response.

The following data is returned in JSON format by the service.

**[contentType](#API_GetModel_ResponseSyntax)**

The content-type for the model.

Type: String

**[description](#API_GetModel_ResponseSyntax)**

The description of the model.

Type: String

**[id](#API_GetModel_ResponseSyntax)**

The identifier for the model resource.

Type: String

**[name](#API_GetModel_ResponseSyntax)**

The name of the model. Must be an alphanumeric string.

Type: String

**[schema](#API_GetModel_ResponseSyntax)**

The schema for the model. For `application/json` models, this should be JSON schema draft 4 model. Do not include "\\\*/" characters in the description of any properties because such "\\\*/" characters may be interpreted as the closing marker for comments in some languages, such as Java or JavaScript, causing the installation of your API's SDK generated by API Gateway to fail.

Type: String
