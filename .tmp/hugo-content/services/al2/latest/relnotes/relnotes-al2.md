December 23, 2021

[Updated NSS to fix CVE-2021-43527.](relnotes-20211201.md)

Amazon Linux 2 2.0.20211201.0 fixed an issue with Network Security Services (NSS) that affected up to and including version 3.73. This issue
caused NSS to be vulnerable to a heap overflow when handling DER-encoded DSA or RSA-PSS signatures.

December 1, 2021

[Amazon Linux 2 .0.20211103.0 repositories point to the Amazon S3 dual stack IPv4/IPv6 endpoint.](relnotes-20211103.md)

The system-release was updated to point the Amazon Linux repositories to the Amazon S3 dual stack IPv4/IPv6 endpoint.

November 3, 2021

[Updated packages](relnotes-20211005.md)

Amazon Linux 2 2.0.20211005.0 includes updated packages for this release.

October 5, 2021

[Update ca-certificates to version 2021.2.50-72.amzn2.0.1](relnotes-20211001.md)

Amazon Linux 2 2 .0.20211001.1 updated the ca-certificates to version 2021.2.50-72.amzn2.0.1. This addresses the expiring IdentTrust DST Root CA
X3. This affected some Let’s Encrypt TLS certificates.

October 1, 2021

[Amazon Linux 2 2.0.20210813.1 Fixed boot problems with grub2 on arm64 and improved compatibility of grub2 generated config file with grubby.](relnotes-20210813.md)

Fixed occasional boot problems with grub2 on ARM64 and improved compatibility of grub2 generated config file with grubby.

August 13, 2021

[Amazon Linux 2 2.0.20210721.2 GRUB updated to 2.06](relnotes-20210721.md)

GRUB has been updated to 2.06 with some launch time improvements.

July 21, 2021

[Amazon Linux 2 2.0.20210525.0 Updated chrony to 4.0.](relnotes-20210525.md)

Updated chrony to 4.0 from 3.5.1.

May 25, 2021

[Amazon Linux 2 2.0.20210427.0 The ec2-net-utils bug was fixed with multiple secondary IPs attached to one ENI.](relnotes-20210427.md)

Amazon Linux 2 2.0.20210427.0 The ec2-net-utils bug was fixed with multiple secondary IPs attached to one ENI.

April 27, 2021

[Amazon Linux 2 2.0.20210421.0 Updated irqbalance to 1.7.0.](relnotes-20210421.md)

Updated irqbalance to 1.7.0 from 1.5.0. AL2 AMIs default to HTTPS for repository access.

April 21, 2021

[Amazon Linux 2 2.0.20210318.0 yum will no longer make IMDSv1 calls.](relnotes-20210318.md)

With this update, yum will no longer attempt to make IMDSv1 calls. The amazon-linux-extras utility has been updated to support a simpler
format of the Extras catalog.

March 18, 2021

[Amazon Linux 2 2.0.20210126.0 Connect to yum over HTTPS.](relnotes-20210126.md)

With this update, Amazon Linux 2 can now connect to its yum repositories over HTTPS. This can be enabled on boot or at runtime.

January 26, 2021

[Amazon Linux 2 2.0.20201218.0 Renewed GPG key.](relnotes-20201218.md)

This update renewed GPG key and updated system-release to allow for the use of HTTPS repositories with Amazon Linux.

December 18, 2020

[Amazon Linux 2 2.0.20201111.0 Fixed time calculation errors when using dates after 2038.](relnotes-20201111.md)

Addressed the glibc bug that lead to time calculation errors whenever using dates after 2038. Instance launch times were also improved.
This was achieved by adding the new dracut-config-ec2 package, which ensure that the initramfs that are built for use inside EC2 don’t
include extra files that aren’t already used inside EC2.

November 11, 2020

[Amazon Linux 2 2.0.20200904.0 Updated to Python 2.7.18.](relnotes-20200904.md)

Two kernel CVEs were added and the CVE-2019-20907 bug in python 2.7.18 was addressed.

September 4, 2020

[Amazon Linux 2 2.0.20200824.0 Security updates for gettext, python2-rsa, and python. Updated AWS CLI.](relnotes-20200824.md)

Included are security updates for gettext, python2-rsa, and python. The AWS CLI was updated, and a bug was addressed for the
amazon-linux-extras utility. After this fix, the utility no longer recommends deprecated topics.

August 24, 2020

[Amazon Linux 2 2.0.20200722.0 Security updates and bug fixes.](relnotes-20200722.md)

Security updates were added for libxml2 and thunderbird, and a dependency bug was fixed for system-rpm-config.

July 22, 2020

[Amazon Linux 2 Python updated to 2.7.18.](relnotes-06-17-2020.md)

Python 2.7 was updated to most recent upstream version - 2.7.18. The ca-certificates fix was addressed for the Sectigo intermediate CA
expiration issue.

June 17, 2020

[Amazon Linux 2 Kernel Live Patching is in public preview.](relnotes-05-29-2020.md)

Amazon Linux 2 added the Kernel Live Patching Public Preview, which applies CVE fixes without the need for reboot.

May 29, 2020

[Amazon Linux 2 Linux kernel 4.19 is available.](relnotes-07-18-2019.md)

Linux kernel 4.19 was made available in Amazon Linux 2 Extras channel.

July 18, 2019

[Amazon Linux 2 Amazon EC2 Instance Connect is enabled by default.](relnotes-06-27-2019.md)

Changed Amazon EC2 Instance Connect is enabled by default.

June 27, 2019

[Amazon Linux 2 AMIs with NVIDIA GPU drivers are available.](relnotes-05-23-2019.md)

Amazon Linux 2 AMIs with NVIDIA GPU drivers are now pre-installed and pre-configured for use on the P and G instance families that are available
on AWS Marketplace.

May 23, 2019

[Amazon Linux 2 A new systemd service is added to launch on boot.](relnotes-05-14-2019.md)

A new systemd service was added to launch on boot. The service submits the host public keys for identity validation to support an
upcoming feature for connecting to EC2 instances.

May 14, 2019

[Incorrect permissions fixed.](relnotes-03-13-2019.md)

This update fixes incorrect permissions for some system files.

March 13, 2019

[A1 support](relnotes-11-26-2018.md)

Amazon Linux 2 is now supported for use on A1 instances.

November 26, 2018

[ENA driver update introduces Low Latency Queues](relnotes-11-19-2018.md)

An ENA driver update was added. This update introduces Low Latency Queues (LLQ) for improved average and tail latencies. It also adds
support for receive checksum offload that improves CPU utilization.

November 19, 2018

[Updated the OpenSSH daemon configuration file.](relnotes-10-31-2018.md)

The OpenSSH daemon configuration file /etc/ssh/sshd\_config has been updated. The AuthorizedKeysCommand value is configured to point to a
customized script and to /opt/aws/bin/curl\_authorized\_keys to support an upcoming feature to read SSH public keysfrom the EC2 instance
metadata during the SSH connection process.

October 31, 2018

[Support for 32-bit Libraries.](relnotes-9-25-2018.md)

Amazon Linux 2 now supports 32-bit libraries and compatibility packages,. With this support, you can now run 32-bit applications on Amazon Linux 2.

September 25, 2018
