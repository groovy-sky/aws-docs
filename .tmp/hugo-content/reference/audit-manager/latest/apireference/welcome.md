# Welcome

Welcome to the AWS Audit Manager API reference. This guide is for developers who
need detailed information about the Audit Manager API operations, data types, and
errors.

Audit Manager is a service that provides automated evidence collection so that you
can continually audit your AWS usage. You can use it to assess the
effectiveness of your controls, manage risk, and simplify compliance.

Audit Manager provides prebuilt frameworks that structure and automate assessments
for a given compliance standard. Frameworks include a prebuilt collection of controls with
descriptions and testing procedures. These controls are grouped according to the
requirements of the specified compliance standard or regulation. You can also customize
frameworks and controls to support internal audits with specific requirements.

Use the following links to get started with the Audit Manager API:

- [Actions](api-operations.md): An
alphabetical list of all Audit Manager API operations.

- [Data types](api-types.md): An alphabetical list of all Audit Manager data
types.

- [Common\
parameters](commonparameters.md): Parameters that all operations can use.

- [Common errors](commonerrors.md):
Client and server errors that all operations can return.

If you're new to AWS Audit Manager, we recommend that you review the [AWS Audit Manager User Guide](../../../../services/audit-manager/latest/userguide/what-is.md).
